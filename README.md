# Evidence Ledger｜実績台帳

自分に関する「事実」「証拠」「条件」「振り返り」を保存し、必要なときだけ面接・職務経歴書用に編集するブラウザアプリです。

## できること

- 受信箱で、1行の事実メモをすぐ保存
- 台帳で実例・能力・習慣・気づきを検索・絞り込み
- 記録ごとに以下を分けて保存
  - 記録の扱い（事実／自己観察／仮説）
  - 事実
  - 証拠（案件名、数値、URL、資料名など）
  - 条件・背景（期限、裁量、関係者、不確実性）
  - 振り返り（好き度、消耗度、扱える範囲、反証、次の実験）
  - 面接・職務経歴書向けの提示用メモ
- 好き度 × 扱える範囲、扱った要素の傾向を確認
- 面接向け・職務経歴書向けの下書きを生成
- TSVで全件バックアップ、絞り込み書き出し、追加、部分更新、全置換・復元
- TSVの反映前に差分表示。取り込み前の状態を自動バックアップし、直前の取り込みを戻す

## 使い方

1. `index.html` をブラウザで開く、または静的ホスティングへ配置する。
2. 最初は「受信箱」に事実を短く入れる。
3. 後から「台帳」で開き、証拠・条件・振り返りを足す。
4. 必要な記録だけを「出力」で選び、面接用の下書きにする。

データはブラウザの IndexedDB に保存されます。端末やブラウザを変える前には、必ず **データ → 全件TSVを保存** を実行してください。


## 初期台帳データ

この版には、これまでの会話・制作履歴をもとにした **65件の個人用初期台帳** を含めています。ヘッダーの「初期台帳を追加」から、既存の記録を消さずに追加できます。

- `sample-data.js`：アプリが読み込む初期データ
- `sample-data.tsv`：スプレッドシートで見直すための編集用データ
- `SAMPLE_DATA.md`：事実／自己観察／仮説の区別とデータの扱い

### 重要：公開GitHub Pagesにはそのまま上げない

初期台帳には、個人のキャリア情報に加え、業務の数値・企画準備・社内外の関係者に関する記録が含まれます。
**このまま公開リポジトリへpushすると、`sample-data.js` と `sample-data.tsv` の内容は誰でも取得できます。**

- 個人用データ入りの版：ローカルまたは非公開環境で管理
- 公開GitHub Pages版：`sample-data.js` と `sample-data.tsv` を、空または公開可能なダミーデータへ置き換える

## GitHub Pagesでの公開

このフォルダの中身をGitHubリポジトリのルートへ置き、GitHub Pages の公開元を `main` ブランチの `/ (root)` に設定します。

サブフォルダ配下へ置く場合も、相対パスで参照しているため、そのまま動作します。

アプリアイコンは `assets/` 配下に置いています。`index.html`、PWAマニフェスト、iPhoneのホーム画面追加用アイコンはいずれも同じ素材を参照するため、GitHubへ上げるときは `assets/` フォルダも必ず含めてください。

## TSVの基本ルール

### 追加

- `タイトル` が必須
- `__id` が空なら自動で新しいIDを発行
- 既存IDが含まれていた場合、既存データは変更しない

### 部分更新

- `__id` が必須
- 列がない、またはセルが空欄の場合は、その値を変更しない
- 値を消すときだけ `__CLEAR__` を入力する

例：タグと面接利用だけを更新する。

```tsv
__id	タグ	面接での利用
既存ID	英語 | リサーチ | 海外事例	候補
```

### 全件復元・全置換

- 現在の台帳をTSVの内容で置き換える
- TSVにない記録は削除される
- 実行前の台帳は自動バックアップされる
- 誤操作防止の確認文が必要

## データの考え方

このツールは自己採点アプリではありません。

- 「好き／嫌い」は、作業そのものではなく条件への反応として記録する
- 「得意」は、抽象的な点数ではなく、どの範囲まで自走できるかで考える
- 「定量／定性」は対立させず、数字・文脈・人・運用・制作・仕組みの複数軸で残す
- 面接で使う文章は、私的な記録や未整理の事実とは別の層で編集する

## ファイル構成

```text
index.html           アプリ本体
styles.css           レスポンシブUI
app.js               IndexedDB、TSV、画面ロジック
sample-data.js       個人用の初期台帳（公開時は要置換）
sample-data.tsv      初期台帳のTSV編集用ソース（公開時は要置換）
SAMPLE_DATA.md       初期台帳の根拠区分と扱い
assets/              アプリアイコン（favicon／PWA／ヘッダー用）
manifest.webmanifest PWA用の基本設定
favicon.svg          旧SVGアイコン（互換用。現在の画面参照はassets配下のPNG）
```
