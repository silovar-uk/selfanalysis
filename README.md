# selfanalysis｜実績台帳

自分の実績・経験・知識を、**事実／証拠／条件／振り返り**に分けて残すための個人用Webアプリです。

「自分は何者か」を採点するのではなく、

> どんな状況で、何を実際に行い、どんな証拠を残し、どの条件で力が出た／出なかったか

を後から検証できる台帳として使います。

## できること

- 30秒で残せる「受信箱」へのクイック記録
- 実例・能力・知識・習慣・仮説・できないこと／制約の蓄積
- 事実／証拠／条件・背景／振り返り／提示用メモの分離
- タグ・領域・記録タイプ・面接利用での検索と絞り込み
- 証拠待ち、条件未整理、振り返り待ちの整理キュー
- 好き度 × 扱える範囲、扱う要素、条件タグの発見画面
- 面接・職務経歴書向けの下書き出力
- TSVの全件出力、絞り込み出力、テンプレートコピー
- TSVの追加／部分更新／全件復元／全置換
- 反映前プレビュー、自動スナップショット、直前操作に戻せる設計
- IndexedDBへのローカル保存（アカウント・サーバー不要）

## 使い方

1. `index.html` をブラウザで開く、または GitHub Pages で公開する。
2. 最初は「受信箱」から、評価せずに記録を積む。
3. 必要なときだけ詳細画面で、証拠・条件・振り返りを足す。
4. まとまった編集は「データ」からTSVで行う。

> ブラウザの `file://` 直開きでも基本的に動くが、モジュール読み込みの都合でローカルサーバーかGitHub Pagesを推奨します。

## GitHub Pagesで公開する

1. GitHubでリポジトリを作成する（例：`selfanalysis`）。
2. このフォルダの中身をリポジトリのルートへアップロードする。
3. GitHubの **Settings → Pages** を開く。
4. **Deploy from a branch** を選び、`main` ブランチの `/(root)` を指定する。
5. 保存後、表示されるURLを開く。

相対パスのみで動くため、プロジェクトページ配下でも利用できます。

## データ保存について

- 実データはブラウザのIndexedDBに保存されます。
- 同じブラウザでも、サイトデータを削除すると消えることがあります。
- 定期的に **データ → 全件バックアップ → TSVを保存** をしてください。
- 別PCへ移すときは、保存したTSVを **全件復元** で取り込みます。

## TSVの重要ルール

### 追加

新規行だけ追加します。既存の `__id` と一致する行は触りません。

### 部分更新

`__id` が一致する既存レコードの、**TSVに含まれる列だけ**を更新します。

- 列がない：変更しない
- セルが空欄：変更しない
- 値がある：その値へ更新
- `__CLEAR__`：その項目を明示的に消去

例：タグと面接利用だけを更新するTSV

```tsv
__id	タグ	面接での利用
sample-work-ticket-segmentation	チケット | CRM | 集客	使う
```

## 初期台帳について

`sample-data.js` と `sample-data.tsv` には、個人用の初期台帳データが入っています。業務の数値、未公表準備、社内情報、連絡先などが含まれる場合は、**公開リポジトリにそのままpushしないでください**。

公開GitHub Pagesにする場合は、以下のどちらかにしてください。

- `sample-data.js` と `sample-data.tsv` をダミーデータへ差し替える
- GitHubリポジトリを非公開運用する

## バージョン規則

配布ZIPは `selfanalysis-v01.zip` を起点に、更新ごとに `selfanalysis-v02.zip`、`selfanalysis-v03.zip`、`selfanalysis-v04.zip`、`selfanalysis-v05.zip` と連番にします。

## v02の更新

- 初期台帳を **94件** に拡張。大きな案件だけでなく、確認依頼、入稿規定、営業日逆算、公開前点検、会場導線、データ定義、配布品質、対人の振り返りなど、日常の小さな実例を29件追加。
- 初期台帳件数の表示を固定値ではなく、`sample-data.js` の件数に連動するように変更。

すでにv01を使っている場合も、更新後のページで「初期台帳を追加」を押すと、既存の記録を残したまま新しい初期データだけ追加できます。

## v03の更新

- 初期台帳を **112件** に拡張。`できないこと・制約` を **18件**追加。
- 「失敗・苦手」を `できないこと・制約` に改称。既存の古い分類も読み込み時に自動で置き換える。
- 記録の入力で、制約は「能力の欠陥」ではなく、前提に置かない条件・必要な支援・代替手段を残すものだと明記。
- 「次に試すこと」を `次に試すこと・代替策` に改め、改善だけでなく、避ける・頼る・仕組みで補う選択肢を記録できるようにした。
- 整理画面に `できないこと・制約` の専用カードとフィルタを追加。

## v04の更新｜できることチェック

- `スキルチェック` を追加。MBTIのようなタイプ診断ではなく、**具体的な操作・行為**を1問ずつ確認します。
- 質問は **218問**。Excel・データ・文章・広報・企画進行・イベント運用・Web／AI・英語・制作進行・働き方の前提までを収録しています。
- 回答は以下の4つだけです。
  - `○ 一人でできる`
  - `△ 見ながらならできる`
  - `× 今はできない`
  - `— 未経験・未確認`
- 自由記入は必須にせず、思い出・根拠・条件が浮かんだときだけ、**その質問に紐づくメモ**として入力できます。メモは入力中に自動保存されます。
- 途中でも、回答済みの質問と質問ごとのメモを **Markdown／テキスト** として、コピーまたはファイル保存できます。
- 回答は通常の台帳レコードとは別に、ブラウザのIndexedDBへ保存されます。
- 質問バンクは `skill-questions.js` に分離しています。増やす・直す・自分向けに入れ替える場合は、このファイルを編集します。

### スキルチェックの使い方

1. ナビゲーションの **スキルチェック** を開く。
2. まずは **おすすめ18問**、または気になる領域から始める。
3. 各質問に○△×—で答える。答えなくても次へ進めます。
4. 根拠や条件を残したい質問だけ、**メモを添える** を押して短く書く。
5. 途中でも **Markdown／テキストを保存** または **コピー** で出力する。

## v05の更新｜答えやすさと途中活用を強化

- 最初の入口を **「3分だけ答える｜6問」** に変更。218問を最初から見せず、短く始められるようにした。
- 「今の仕事から始める」を追加。`数字と判断`／`言葉と発信`／`企画と調整`／`現場と制作` の4コースから入れる。
- 回答テンポを選べるようにした。
  - `サクサク`：回答後に次の質問へ進む
  - `じっくり`：その場に留まり、条件やメモを残す
- サクサク回答でも、画面上に **直前の回答** を残し、あとからメモ追加・回答の取り消しができる。
- `△` を選んだとき、必要に応じて `見本があれば`／`検索すれば`／`手順書があれば`／`レビューがあれば` をワンタップで記録できる。
- 各質問に、自由記入なしでも残せるピンを追加。
  - `仕事で使った`
  - `個人で試した`
  - `条件あり`
  - `証拠を後で足す`
  - `学びたい`
- 回答数に応じて、カテゴリ別の **途中の見取り図** を表示。タイプ診断ではなく、○・△・ピン・メモが付いた具体操作をそのまま見せる。
- Markdown／テキスト出力にフィルタを追加。`すべて`／`○だけ`／`○＋△`／`メモ・ピンあり`／`×・制約` で、途中でも必要な情報だけコピー・保存できる。

### v05の使い方

1. **スキルチェック**を開き、まずは **3分だけ答える｜6問** を押す。
2. 迷う質問は `△` や `—` で止めずに先へ進む。
3. 思い出が出たら、その質問にピンまたは短いメモを添える。
4. 10問前後から、**今のところ見えていること** を確認する。
5. 面接や職務経歴書の下書きに使いたいときは、`○＋△` や `メモ・ピンあり` でMarkdownを出力する。
